# Phase 2 -- UI Design Contract

> Visual and interaction contract for the EPUB Reader phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (React Native / NativeWind) |
| Preset | not applicable |
| Component library | none (custom RN components with NativeWind) |
| Icon library | expo/vector-icons (SF Symbols via IconSymbol on iOS, MaterialIcons fallback) |
| Font | System default (system-ui on iOS, Roboto on Android) -- matches existing `Fonts` constant in `constants/theme.ts` |

**Note:** This is a React Native / Expo project using NativeWind (Tailwind CSS for RN). shadcn is not applicable. The design system is defined through NativeWind utility classes and the existing `Colors` and `Fonts` constants.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, icon-to-label gaps |
| md | 16px | Default element spacing, list item padding |
| lg | 24px | Section padding, screen horizontal margins |
| xl | 32px | Reader toolbar height padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets: minimum 44px height/width per iOS HIG and Android accessibility guidelines
- Reader content area: full-bleed (0px horizontal padding) -- epub.js WebView manages its own internal margins
- Safe area insets: handled by `react-native-safe-area-context`, not fixed values

## Typography

### App Chrome (outside reader)

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 600 (semibold) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 700 (bold) | 1.2 |

### Reader Content (inside epub.js WebView)

Reader typography is controlled by epub.js `changeTheme()` / `changeFontSize()` API. The mobile app exposes user controls for:

| Setting | Default | Range | Step |
|---------|---------|-------|------|
| Font size | 100% (1em) | 80% to 150% | 10% |
| Font family | System serif | System serif, system sans-serif | -- |

Font size persists per-user (stored locally, synced later in Phase 5).

## Color

### App Chrome (screens, navigation, toolbars)

Uses the existing `Colors` constant from `constants/theme.ts`, respecting system light/dark mode:

**Light Mode:**

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFFFFF | Screen backgrounds, surfaces |
| Secondary (30%) | #F3F4F6 (gray-100) | Cards, grouped list backgrounds, toolbar bg |
| Accent (10%) | #0A7EA4 | Import CTA button, active tab icon, TOC current chapter indicator |
| Destructive | #DC2626 (red-600) | Not used in this phase (no delete actions) |
| Text primary | #11181C | Headings, body text |
| Text secondary | #687076 | Subtitles, metadata, icon default |

**Dark Mode:**

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #151718 | Screen backgrounds, surfaces |
| Secondary (30%) | #1F2937 (gray-800) | Cards, grouped list backgrounds, toolbar bg |
| Accent (10%) | #FFFFFF | Import CTA button, active tab icon, TOC current chapter indicator |
| Destructive | #EF4444 (red-500) | Not used in this phase |
| Text primary | #ECEDEE | Headings, body text |
| Text secondary | #9BA1A6 | Subtitles, metadata, icon default |

Accent reserved for: Import Book button fill, active tab icon, TOC current-chapter highlight bar, loading spinner tint.

### Reader Themes (inside epub.js WebView)

Ported directly from the desktop app's existing theme definitions:

| Theme Name | Background | Text Color | Source |
|------------|-----------|------------|--------|
| Light | #FFFFFF | #000000 | `whiteReaderTheme.ts` |
| Dark | rgb(48, 48, 50) / #303032 | rgb(184, 184, 185) / #B8B8B9 | `grayReaderTheme.ts` |
| Sepia | rgb(246, 240, 226) / #F6F0E2 | rgb(43, 42, 40) / #2B2A28 | `yellowReaderTheme.ts` |

These are applied via `@epubjs-react-native/core` `changeTheme()` API. Theme names match the desktop `ThemeType` enum values: `white`, `dark`, `yellow`.

## Screens and Interactions

### Screen 1: Library (Home Tab -- replaces current placeholder)

**Layout:** Vertical scrolling list of imported books. Each row shows book cover thumbnail (left), title + author (right).

| Element | Specification |
|---------|--------------|
| Empty state | Centered vertically. Icon + heading + body + CTA button. |
| Book row height | 80px minimum, with 16px vertical padding |
| Cover thumbnail | 48px wide, aspect ratio preserved, rounded-md (8px radius) |
| Book title | 16px, weight 600, single line with ellipsis truncation |
| Book author | 14px, weight 400, text-secondary color, single line with ellipsis truncation |
| Row tap target | Full row is tappable, navigates to Reader screen |
| Pull to refresh | Not applicable (no sync in this phase) |

**States:**
- Empty (no books imported)
- Populated (1+ books)
- Loading (book being imported -- show ActivityIndicator inline)

### Screen 2: Reader (Full-screen, pushed onto stack)

**Layout:** Full-screen WebView rendering the EPUB via `@epubjs-react-native/core`. Toolbar overlays at top, hidden by default, shown on single tap.

| Element | Specification |
|---------|--------------|
| WebView | Full screen, edge-to-edge. Safe area insets applied to toolbar only, not content. |
| Navigation | Swipe left/right for page turn (built into @epubjs-react-native). |
| Toolbar (top) | Semi-transparent overlay. Back button (left), book title (center, truncated), theme/font button (right). |
| Toolbar toggle | Single tap on reading area toggles toolbar visibility. |
| Toolbar auto-hide | Toolbar hides after 3 seconds of no interaction. |
| Page indicator | Bottom center, small text: "Page X of Y" or chapter progress percentage. Matches reader theme colors. |

**Toolbar buttons (right cluster):**

| Button | Icon | Action |
|--------|------|--------|
| TOC | list icon | Opens TOC bottom sheet |
| Theme | palette icon | Opens theme/font bottom sheet |
| Back | chevron-left | Navigates back to Library |

### Sheet 1: Table of Contents (Bottom Sheet)

**Layout:** Bottom sheet (50% screen height default, draggable to 90%). Flat list of chapter titles.

| Element | Specification |
|---------|--------------|
| Sheet handle | 4px height, 36px width, centered, rounded-full, gray-400 |
| Chapter row | 48px height, 16px horizontal padding |
| Chapter title | 16px, weight 400. Current chapter: weight 600 + accent color left border (3px) |
| Nested chapters | Indented 24px per nesting level |
| Tap action | Navigate to chapter via `goToLocation()`, dismiss sheet |

### Sheet 2: Theme and Font Controls (Bottom Sheet)

**Layout:** Bottom sheet (auto-height, not scrollable). Two sections: theme picker, font size slider.

| Element | Specification |
|---------|--------------|
| Theme picker | Row of 3 circular swatches (40px diameter): Light (#FFF border), Dark (#303032 fill), Sepia (#F6F0E2 fill). Active swatch has 2px accent-color ring. |
| Font size control | Row with minus button, current size label ("100%"), plus button. Buttons are 44px touch targets. |
| Font family toggle | Two-segment control: "Serif" / "Sans". 44px height, rounded-lg. |

### Screen 3: File Import (system picker, not a custom screen)

**Trigger:** "Import Book" button on Library screen (FAB or inline CTA in empty state).

| Element | Specification |
|---------|--------------|
| Import button (empty state) | Full-width, accent background, white text, rounded-lg, 48px height |
| Import button (populated state) | FAB (floating action button), bottom-right, 56px diameter circle, accent background, plus icon in white |
| File picker | System `expo-document-picker` with MIME filter `application/epub+zip` |
| Import progress | Replace FAB with circular ActivityIndicator during file copy. Return to FAB on completion. |
| Import success | Book appears at top of library list. No toast or alert -- the new entry is the confirmation. |
| Import failure | Inline error banner at top of Library: red-50 background, red-600 text, dismiss X button. |

## Interaction Contracts

### Page Turn Animation

Use `@epubjs-react-native` built-in swipe gesture handling. No custom animation overlay needed (unlike desktop which uses framer-motion). The WebView handles page transitions internally.

### Position Persistence

| Event | Action |
|-------|--------|
| Page turn (locationChanged callback) | Debounce 500ms, write ePubCFI to expo-sqlite |
| App background | Save current ePubCFI immediately (AppState listener) |
| Book reopen | Pass saved ePubCFI as `initialLocation` prop |
| No saved position | Open at beginning (ePubCFI not set) |

### Theme Switching

| Event | Action |
|-------|--------|
| User taps theme swatch | Call `changeTheme()` on reader instance. Update toolbar colors to match. Persist choice to local storage. |
| Book open | Apply last-used theme from local storage. Default: follows system light/dark preference (Light theme for light mode, Dark theme for dark mode). |

### Font Size Change

| Event | Action |
|-------|--------|
| User taps +/- | Call `changeFontSize()`. Clamp to 80%-150% range. Persist to local storage. |
| Book open | Apply last-used font size from local storage. Default: 100%. |

## Accessibility

| Concern | Specification |
|---------|--------------|
| Touch targets | Minimum 44x44px for all interactive elements |
| Reader content | epub.js WebView inherits platform VoiceOver/TalkBack support for rendered HTML |
| Color contrast | All text meets WCAG AA (4.5:1 for body, 3:1 for large text). Verified: Light theme #000 on #FFF = 21:1, Dark theme #B8B8B9 on #303032 = 5.9:1, Sepia #2B2A28 on #F6F0E2 = 11.7:1 |
| Screen reader labels | All icon-only buttons have `accessibilityLabel` props |
| Reduce motion | Respect `useReducedMotion()` -- disable page turn animations if system prefers reduced motion |
