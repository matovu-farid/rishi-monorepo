# Phase 11 — UI Design Contract

> Visual and interaction contract for Phase 11: Mobile Feature Parity. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|

| Preset | not applicable |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, border gaps |
| md | 16px | Default element spacing, component padding |
| lg | 24px | Section padding, card internal padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Touch targets use 44px (w-11 h-11) per iOS Human Interface Guidelines. The realtime voice FAB uses 56px diameter for primary affordance.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (text-base) | 400 (normal) | 1.5 |
| Caption | 14px (text-sm) | 400 (normal) | 1.43 |
| Heading | 20px (text-xl) | 600 (font-semibold) | 1.2 |
| Label | 12px (text-xs) | 600 (font-semibold) | 1.33 |

Weights used: 400 (normal) for Body and Caption, 600 (font-semibold) for Heading and Label. A 12px/600 label is visually distinct from 16px/400 body text without needing an intermediate weight.

Source: Extracted from existing mobile components (LibraryEmptyState, ChatMessage, ReaderToolbar, VoiceMicButton).

## Interaction Contracts

### RealtimeVoiceButton

**Placement:** Floating action button in the reader toolbar area OR in the chat screen header. Positioned bottom-right when used as FAB (16px from right edge, 16px above tab bar).

**States:**

| State | Visual | Behavior |
|-------|--------|----------|
| Idle | 56px circle, `#0a7ea4` background, `waveform` icon in white | Tap to start realtime session |
| Connecting | 56px circle, `#0a7ea4` background, `ActivityIndicator` white | WebRTC handshake in progress; disable tap |
| Active (listening) | 56px circle, `#0a7ea4` background with pulse animation (opacity 0.6-1.0, 800ms cycle), `waveform` icon in white | AI is listening; tap to end session |
| Active (AI speaking) | 56px circle, `#0a7ea4` background, `waveform` icon in white with scale animation | AI audio playing through speaker; tap to end session |
| Ending | 56px circle, gray-200/gray-700 background, `ActivityIndicator` | Session closing; disable tap |

**Animation:** Reuse the pulse animation pattern from existing `VoiceMicButton` component (react-native-reanimated `withRepeat` + `withTiming`).

**Accessibility:**
- `accessibilityLabel`: "Start voice conversation" (idle), "Voice conversation active, tap to end" (active), "Connecting" (connecting)
- `accessibilityRole`: "button"
- `accessibilityState`: `{ disabled: true }` when connecting or ending

### SyncStatusIndicator

**Placement:** Library screen header area, right-aligned. Compact pill format (not a full-width bar).

**States:**

| State | Icon | Label | Color |
|-------|------|-------|-------|
| not-synced | `arrow.2.circlepath` | "Not synced" | `#9BA1A6` (muted) |
| syncing | `arrow.2.circlepath` (rotating via `withRepeat` animation) | "Syncing..." | `#0a7ea4` (accent) |
| synced | `checkmark.circle.fill` | "Synced" | `#22C55E` (green-500) |
| error | `exclamationmark.triangle.fill` | "Sync failed" | `#F59E0B` (amber-500) |
| offline | `wifi.slash` | "Offline" | `#9BA1A6` (muted) |

**Layout:** Horizontal row: `[icon 16px] [4px gap] [label text-xs font-semibold] [4px gap] [relative time text-xs text-muted]`

**Tap behavior:**
- `synced`, `not-synced`, `error`: Tap triggers manual sync
- `syncing`, `offline`: Tap disabled (no action)

**Relative time:** Displayed after label in muted text. Format: "Just now", "2m ago", "1h ago", "1d ago". Shown only when `lastSyncAt` is non-null.

**Accessibility:**
- `accessibilityLabel`: "{label} - Last synced: {relative time}" or "{label} - Never synced"
- `accessibilityRole`: "button" when tappable, "text" when not

### GuardrailWarning

**Trigger:** Displayed when the server-side tripwire classification returns `tripwireTriggered: true` for an AI realtime response.

**Visual:** Inline banner below the reader toolbar or at the top of the active voice session overlay.

| Property | Value |
|----------|-------|
| Background | `#FEF3C7` light (amber-100) / `#78350F` dark (amber-900) |
| Border | 1px `#F59E0B` (amber-500) |
| Border radius | 8px |
| Padding | 16px horizontal, 8px vertical |
| Icon | `exclamationmark.triangle.fill` 16px, `#F59E0B` |
| Text | 14px (text-sm), normal weight |
| Layout | `[icon] [8px] [text]` horizontal row |

**Copy:** "The AI went off-topic. Redirecting back to your book."

**Behavior:** Auto-dismisses after 4 seconds with a FadeOut animation (200ms). No manual dismiss button needed.

**Accessibility:**
- `accessibilityRole`: "alert"
- `accessibilityLiveRegion`: "assertive"

## Platform-Specific Notes

### iOS
- Microphone permission: `NSMicrophoneUsageDescription` in Info.plist (already present from Phase 7 STT)
- Safe area insets: All overlays respect `useSafeAreaInsets()` (existing pattern)

### Android
- Microphone permission: `RECORD_AUDIO` in AndroidManifest.xml (already present from Phase 7 STT)
- Icon fallbacks for new SF Symbols:
  - `phone.fill` -> `phone` (MaterialIcons)
  - `arrow.2.circlepath` -> `sync` (MaterialIcons)
  - `checkmark.circle.fill` -> `check-circle` (MaterialIcons)
  - `exclamationmark.triangle.fill` -> `warning` (MaterialIcons)
  - `wifi.slash` -> `wifi-off` (MaterialIcons)
  - `waveform` -> `graphic-eq` (MaterialIcons)
