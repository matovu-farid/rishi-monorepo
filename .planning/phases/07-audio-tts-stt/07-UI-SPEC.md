## Component Inventory

### New Components

#### 1. TTSControls

Floating bottom bar for TTS playback. Rendered inside the EPUB/PDF reader screen when TTS is active.

**Layout:** Horizontal row, centered at bottom of screen with `bottom: insets.bottom + 16`. Full-width minus 32px horizontal margin. Rounded corners (rounded-2xl). Background: `bg-black/80` with backdrop blur (matching desktop pattern). Height: 56px.

**Controls (left to right):**
1. **Previous** button (skip back icon) -- 44x44px touch target
2. **Play/Pause** button (toggles between play.fill / pause.fill) -- 44x44px touch target
3. **Next** button (skip forward icon) -- 44x44px touch target
4. **Stop** button (stop.fill icon) -- 44x44px touch target

**States:**
- `idle` -- not visible (unmounted)
- `loading` -- play button shows ActivityIndicator (white, size 20), other buttons disabled (opacity 0.3)
- `playing` -- pause icon shown, all buttons enabled, white icons
- `paused` -- play icon shown, all buttons enabled, white icon at 0.8 opacity

**Progress indicator:** Below the controls row, a thin 2px progress bar showing `currentChunkIndex / totalChunks`. Bar color: `#0a7ea4`. Background: `rgba(255,255,255,0.2)`.

**Animation:** Enters from bottom with `SlideInDown.duration(250)`, exits with `SlideOutDown.duration(200)` (react-native-reanimated).

**Accessibility:**
- `accessibilityLabel` on each button: "Previous passage", "Play", "Pause", "Next passage", "Stop reading"
- `accessibilityRole="button"` on each control
- Progress bar: `accessibilityLabel="Reading passage {current} of {total}"`

#### 2. VoiceMicButton

Microphone button added to the existing ChatInput component, positioned to the left of the send button.

**Layout:** 40x40px rounded-full button (`w-10 h-10 rounded-full items-center justify-center`). Margin: `ml-2` from text input, `mr-2` before send button.

**Icon:** `mic.fill` (SF Symbol) / `mic` (MaterialIcons), size 20.

**States:**
- `idle` -- background `bg-gray-200 dark:bg-gray-700`, icon color `#9BA1A6`
- `recording` -- background `bg-red-500`, icon color `#FFFFFF`, pulsing opacity animation (0.6 to 1.0, 800ms cycle, react-native-reanimated)
- `transcribing` -- ActivityIndicator (white, size 16) replaces icon, background `#0a7ea4`
- `disabled` -- same as idle, opacity 0.5

**Interaction:**
- Single tap to start recording
- Single tap again to stop recording and begin transcription
- NOT press-and-hold (simpler, avoids gesture conflicts with keyboard)

**Permission flow:** On first tap, if microphone permission not granted, request via `Audio.requestPermissionsAsync()`. If denied, show inline text below input: "Microphone access required for voice input" in `text-sm text-red-500`.

**Accessibility:**
- `accessibilityLabel`: "Start voice input" (idle), "Stop recording" (recording), "Transcribing" (transcribing)
- `accessibilityRole="button"`
- `accessibilityState={{ disabled: isDisabled }}`

#### 3. TTSToolbarButton

New button added to the existing ReaderToolbar, positioned after the chat button.

**Layout:** 44x44px (`w-11 h-11 items-center justify-center`), same as all other toolbar buttons.

**Icon:** `speaker.wave.2.fill` (SF Symbol) / `volume-up` (MaterialIcons), size 22. Color: `theme.toolbarText` (matches other toolbar icons).

**States:**
- `idle` -- standard toolbar icon appearance
- `active` (TTS playing/paused) -- icon tinted `#0a7ea4` to indicate TTS is active

**Behavior:** Tap toggles TTS controls visibility. If no chunks exist for the book, starts chunking first (shows brief loading state on the button via ActivityIndicator replacing the icon, same size).

**Accessibility:**
- `accessibilityLabel`: "Read aloud" (idle), "Reading aloud" (active)
- `accessibilityRole="button"`

## Interaction Patterns

### TTS Playback Flow

1. User taps `speaker.wave.2.fill` in ReaderToolbar
2. If chunks do not exist: show ActivityIndicator on button, run `ensureBookChunked`, then proceed
3. TTSControls slide up from bottom (SlideInDown)
4. Auto-starts playing from chunk 0 (or last played chunk if resuming)
5. User controls: play/pause (toggle), previous (go back one chunk), next (skip forward one chunk), stop (dismiss controls)
6. Stop dismisses TTSControls (SlideOutDown) and resets playback state

### Voice Input Flow

1. User taps VoiceMicButton in ChatInput
2. If first use: permission dialog shown
3. Button turns red with pulse animation; placeholder becomes "Listening..."
4. User taps again to stop
5. Button shows transcribing state (ActivityIndicator); placeholder becomes "Transcribing..."
6. Transcribed text populates TextInput
7. User reviews text and taps send (or edits first)

### Error Recovery

- **TTS network error:** Show toast-style alert at top of screen: "Unable to play audio. Check your connection and try again." Auto-dismiss after 4 seconds. TTS controls remain visible; user can retry by tapping play.
- **STT network error:** Show inline text below ChatInput: "Could not transcribe audio. Try speaking again." Clears on next recording attempt.
- **Permission denied:** Show inline text below ChatInput: "Microphone access required for voice input." Persists until user grants permission in settings.
