In apps/mobile/lib/sync/engine.ts, add an updatedAt comparison guard in the pull loop for existing non-dirty highlights (around lines 227-243).

Currently the code at line 227 unconditionally calls db.update(highlights).set({...}) for existing non-dirty local highlights. Add a guard so the update only happens if the remote highlight's updatedAt is greater than or equal to the local highlight's updatedAt.

Specifically, after the `if (local.isDirty) continue` check (line 225), add:

```typescript
      // LWW guard: only apply remote update if remote is newer or equal
      const remoteUpdatedAt = (r.updatedAt as number) ?? 0
      if (remoteUpdatedAt < (local.updatedAt ?? 0)) continue
```

Then proceed with the existing db.update call. This ensures that a local highlight that was pushed (and thus is no longer dirty) but has a newer updatedAt than the incoming remote data will not be overwritten by stale remote data.

The guard uses `<` (skip if remote is older) rather than `<=` (skip if equal) because equal timestamps should still apply the remote version to pick up any syncVersion updates from the server.
