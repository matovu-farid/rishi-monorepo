<objective>
Build the TTS (text-to-speech) playback system for the mobile reader. Users tap a speaker icon in the reader toolbar, and the app reads book content aloud by fetching sequential text chunks from the local SQLite chunks table, sending each chunk to the Worker /api/audio/speech endpoint (OpenAI TTS), writing the returned mp3 audio to a temp file, and playing it via expo-audio. A floating control bar provides play/pause/stop/skip controls with a progress indicator.

Purpose: Enables listening to book content — the core audio feature for the reading app.
Output: TTS queue service, player wrapper, React hook, floating controls component, reader toolbar integration.
</objective>

    **Create apps/mobile/hooks/useTTSPlayer.ts:**
    Custom hook that orchestrates TTSQueue and TTSPlayer together:
    ```typescript
    export function useTTSPlayer(bookId: string, filePath: string, format: 'epub' | 'pdf')
    ```
    Returns:
    - `status`: 'idle' | 'loading' | 'playing' | 'paused'
    - `currentChunkIndex`: number
    - `totalChunks`: number
    - `play()`: starts/resumes TTS from current chunk. On first call, calls ensureBookChunked, loads chunks, requests audio for chunk 0, plays it. On subsequent calls, resumes from paused state.
    - `pause()`: pauses current playback
    - `stop()`: stops playback, resets to idle, unmounts TTSControls
    - `next()`: skips to next chunk
    - `previous()`: goes back one chunk
    - `isActive`: boolean (status !== 'idle')

    Uses useRef to hold TTSQueue and TTSPlayer instances. Uses useState for reactive status/progress. Sets up onPlaybackFinished callback to auto-advance to next chunk and prefetch next 2. Cleans up temp audio files (FileSystem.deleteAsync) after each chunk finishes playing. Cleans up on unmount.
