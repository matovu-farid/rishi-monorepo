## Cross-Phase Integration

### Broken Integrations (3)

1. **Desktop Chat → Worker /api/text/completions**
   - `useChat.ts` sends `{ messages }` but Worker reads `{ input }`
   - `useChat.ts` reads `data.message` but Worker returns plain string
   - **Fix:** Align body key and response parsing in useChat.ts

2. **Mobile Realtime → Worker /api/realtime/client_secrets**
   - `session.ts` expects `{ client_secret: { value } }` JSON
   - Worker returns `c.text(parsedResponse.value)` (plain text)
   - **Fix:** Change Worker to return JSON or update session.ts parsing

3. **Mobile Guardrails → Worker /api/text/completions**
   - `guardrails.ts` calls `response.text()` on a `c.json()` response
   - Regex match for JSON object fails; guardrail always fails-open
   - **Fix:** Use `response.json()` then parse, or dedicated endpoint
