## Component Inventory

### 1. SelectionPopover

**Trigger:** User selects text in the EPUB reader (existing `handleTextSelected` callback)
**Component:** shadcn Popover positioned at selection coordinates
**Contents:** Row of 4 color dot buttons (28px diameter each, 8px gap), each filled with the highlight color hex
**Interaction:** Click a color dot to create a highlight in that color. Popover dismisses on selection.
**Keyboard:** Arrow keys navigate dots, Enter confirms, Escape dismisses

### 2. HighlightsPanel

**Trigger:** Toolbar button (lucide `Highlighter` icon) in the reader top bar
**Component:** shadcn Sheet, side="right", width 400px
**Header:** "Highlights" heading (18px semibold) + close button (lucide `X`)
**Body:** ScrollArea containing highlight rows. Each row:
- Left border 3px in highlight color
- Text snippet (14px, max 2 lines, ellipsis overflow)
- Chapter label below snippet (12px, muted-foreground)
- Note preview if present (12px, italic, muted-foreground, max 1 line)
- Click row to navigate reader to that CFI location
- Hover shows edit (lucide `Pencil`, 16px) and delete (lucide `Trash2`, 16px) icon buttons at row right
**Empty state:** Centered in panel body
**Footer:** Highlight count label (12px, muted-foreground): "{n} highlights"

### 3. NoteEditor

**Trigger:** Click edit button on a highlight row in HighlightsPanel
**Component:** shadcn Dialog
**Header:** "Edit Note" (18px semibold)
**Body:** shadcn Textarea, 4 rows, placeholder "Add a note..."
**Footer:** "Cancel" button (variant ghost) + "Save" button (variant default)
**Keyboard:** Cmd+Enter saves, Escape cancels

### 4. ReaderSettings

**Trigger:** Toolbar button (lucide `Settings` icon) in the reader top bar
**Component:** shadcn Popover, align="end", width 280px
**Contents:**
- **Font Size** label (12px medium) + value display (e.g. "1.2em")
  - shadcn Slider, min 0.8, max 2.0, step 0.1, default 1.2
  - Reset button (lucide `RotateCcw`, 14px) appears when value is not 1.2
- Separator (8px vertical margin)
- **Font Family** label (12px medium)
  - Two toggle buttons side by side: "Serif" (Georgia preview) / "Sans" (Inter preview)
  - Active button uses variant="default", inactive uses variant="outline"
**Persistence:** Settings saved to Tauri store (`reader-settings`) on change. Loaded on reader mount.

### 5. ChatPanel

**Trigger:** Toolbar button (lucide `MessageSquare` icon) in the reader top bar
**Component:** shadcn Sheet, side="right", width 440px
**Header:** Book title (14px, truncated) + close button
**Body:** ScrollArea with message list
- User messages: right-aligned, `var(--primary)` background, `var(--primary-foreground)` text, rounded-lg, max-width 85%
- AI messages: left-aligned, `var(--secondary)` background, `var(--secondary-foreground)` text, rounded-lg, max-width 85%
- Source chips below AI messages: row of Badge components (variant="outline"), each showing "Ch. {n}" or truncated chapter name, clickable to navigate reader to source location
- Loading state: 3-dot pulse animation in an AI-aligned bubble
**Footer:** ChatInput component (see below)
**Empty state:** Centered in panel body

### 6. ChatInput

**Component:** Custom component in chat panel footer
**Layout:** Row with text input (shadcn Input, flex-grow) + voice button + send button
- Voice button: lucide `Mic` icon (variant="ghost"), 36px square
- Send button: lucide `SendHorizontal` icon (variant="default"), 36px square, disabled when input is empty and not recording
**Voice recording state:** Mic button background pulses with `var(--destructive)` at 0.2 opacity; recording duration shown as "0:03" label replacing input placeholder
**Keyboard:** Enter sends message (unless Shift held), Escape clears input

### 7. SourceChip

**Component:** shadcn Badge, variant="outline"
**Content:** Truncated source label (max 20 chars), e.g. "Ch. 3: The Garden..."
**Interaction:** Click navigates reader to the source CFI location
**Hover:** Tooltip showing full chapter name and text preview (first 100 chars of chunk)

## Interaction States

### Highlight Creation Flow
1. User selects text in EPUB reader
2. SelectionPopover appears at selection position with 4 color dots
3. User clicks a color dot
4. Highlight is applied to the text (epub.js annotation with fill color + 0.3 opacity + multiply blend)
5. Highlight is saved to local DB via `saveHighlight`
6. `triggerSyncOnWrite()` fires (2-second debounce)
7. SelectionPopover dismisses

### Voice Input Flow
1. User clicks Mic button in ChatInput
2. Browser requests microphone permission (if not already granted)
3. MediaRecorder starts capturing audio (webm/opus)
4. Mic button pulses red, input placeholder shows "Recording... 0:03"
5. User clicks Mic button again to stop
6. Audio blob POSTed to `/api/audio/transcribe`
7. Transcript injected into text input
8. User can edit transcript before sending

### Chat Message Flow
1. User types or voice-inputs a question, presses Enter or clicks Send
2. User message appears right-aligned immediately (optimistic)
3. Loading bubble appears left-aligned (3-dot pulse)
4. RAG retrieval: `get_context_for_query` fetches relevant chunks
5. LLM call: chunks + question sent to `/api/text/completions`
6. AI response replaces loading bubble, source chips appear below
7. Message saved to local DB
8. `triggerSyncOnWrite()` fires
