This means `response.json()` yields the raw LLM output string (e.g., `'{ "isRelevantToBook": true, "isSmallTalk": false }'`), while `response.text()` yields a double-quoted JSON string (e.g., `'"{ \\"isRelevantToBook\\": true }"'`) which breaks the regex.

Change line 29 from:

```typescript
const text = await response.text()
```

to:

```typescript
const text = await response.json() as string
```

The variable name `text` stays the same so the rest of the function (the regex match on line 31 and JSON.parse on line 34) continues to work unchanged. The only difference is that `text` now contains the unwrapped LLM output string instead of a JSON-encoded quoted string, so `text.match(/\{[\s\S]*\}/)` will correctly find the JSON object in the LLM response.
