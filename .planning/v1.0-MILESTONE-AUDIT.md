## Requirement Gaps

### READ-06: PDF page navigation via thumbnails
- **Status:** unsatisfied
- **Phase:** 3
- **Evidence:** PDF page navigation exists (page numbers, slider) but thumbnails were intentionally descoped to v2. REQUIREMENTS.md checkbox is unchecked (`[ ]`). Traceability shows "Pending".
- **Impact:** Low — core PDF reading works; thumbnails are a v2 enhancement

### RAG-08: Server-side embedding fallback for bulk imports
- **Status:** partial
- **Phase:** 6
- **Evidence:** Worker `/api/embed` endpoint exists and works. Client wrapper `server-fallback.ts` with `embedTextsOnServer()` exists. However, `pipeline.ts` only calls `embedBatch` from `embedder.ts` — if on-device model is not downloaded, `embedBook` throws `'Embedding model not ready'` rather than falling back to the server endpoint. The fallback is available as a standalone utility but not automatically triggered.
- **Impact:** Medium — fresh installs where user hasn't downloaded the ~80MB model cannot embed books until model download completes

## Integration Gaps

### Gap 1: Chat screen uses sync-unaware book lookup
- **File:** `apps/mobile/app/chat/[bookId].tsx` line 80
- **Current:** `getBookById(bookId)` — synchronous, no R2 download
- **Required:** `getBookForReading(bookId)` — async, triggers R2 download for synced books
- **Impact:** Books synced from another device have empty `filePath`. Embedding pipeline receives empty path, chunker returns [], vector store is empty, RAG returns no context.
- **Affects:** RAG-01, RAG-04, RAG-05, AUD-01 (for synced books only)

### Gap 2: Desktop first-sync error on fresh install (minor)
- **File:** `apps/main/src/modules/sync-triggers.ts` line 107
- **Issue:** `initDesktopSync()` fires `triggerSync()` immediately on mount before user has logged in
- **Impact:** SyncStatusIndicator shows "Error" on fresh installs. Recoverable after login.
- **Affects:** DSYNC-02 (minor UX)

## Broken E2E Flow

### Desktop Import → Sync → Mobile Chat/RAG
1. ✓ Desktop imports book, hashes, uploads to R2
2. ✓ Desktop sync pushes metadata to D1
3. ✓ Mobile pulls from D1, book appears in library
4. ✓ User opens book in reader (uses `getBookForReading` — works)
5. ✗ User opens Chat for synced book — `getBookById` returns empty `filePath`
6. ✗ `embedBook("", "epub")` — chunker fails silently, no vectors stored
7. ✗ RAG query finds nothing — AI answer has no book context

**Fix:** Replace `getBookById` with `await getBookForReading(bookId)` in `chat/[bookId].tsx`
