Create apps/mobile/app/reader/pdf/[id].tsx -- a PDF reader screen using react-native-pdf.

The screen structure:
```typescript
import { useCallback, useEffect, useRef, useState } from 'react'
import { View, Text, TouchableOpacity, AppState, AppStateStatus, Dimensions } from 'react-native'
import { SafeAreaView } from 'react-native-safe-area-context'
import { useLocalSearchParams, useRouter } from 'expo-router'
import Pdf from 'react-native-pdf'
import { IconSymbol } from '@/components/ui/icon-symbol'
import { getBookById, updateBookPage } from '@/lib/book-storage'
import { Book } from '@/types/book'
```

Component logic:
1. Load book from DB using `getBookById(id)` in useEffect (same pattern as EPUB reader)
2. Track `currentPage` in a ref (for save-on-background) and state (for display)
3. Track `totalPages` from Pdf's `onLoadComplete` callback
4. Save page position:
   - On page change: debounce 500ms, call `updateBookPage(book.id, page)`
   - On app background: save immediately via AppState listener (same pattern as EPUB reader)
   - On back navigation: save before router.back()

PDF component props:
```tsx
<Pdf
  source={{ uri: book.filePath }}
  page={book.currentPage || 1}
  horizontal={true}
  enablePaging={true}
  onLoadComplete={(numberOfPages) => setTotalPages(numberOfPages)}
  onPageChanged={(page) => handlePageChange(page)}
  onError={(error) => console.error('PDF error:', error)}
  style={{ flex: 1, width: Dimensions.get('window').width }}
  trustAllCerts={false}
/>
```

Toolbar (simple, custom for PDF -- do NOT reuse EPUB ReaderToolbar since it has theme/TOC buttons):
- Top bar: back button (left), book title (center, truncated), page indicator (right: "Page X of Y")
- Toggle visibility on tap (same pattern as EPUB reader: tap on reading area toggles toolbar)
- Auto-hide after 3 seconds
- Style: semi-transparent black background (`bg-black/70`), white text

Page navigation controls (bottom bar, visible when toolbar is visible):
- Previous page button (chevron-left icon, 44px touch target)
- Current page text ("X / Y")
- Next page button (chevron-right icon, 44px touch target)
- Go to page: tapping the "X / Y" text opens a simple Alert.prompt (iOS) or text input to jump to a page number. Use `pdfRef.current?.setPage(pageNumber)` -- Note: react-native-pdf does NOT have a setPage method on the ref. Instead, control the `page` prop via state. When user enters a page number, update the `page` state variable and the Pdf component will navigate.

Back button: save current page, then router.back().

The screen must use SafeAreaView from 'react-native-safe-area-context' (NOT from react-native).

Do NOT add thumbnail navigation in this task -- page number display and go-to-page input satisfy READ-06 for MVP. Thumbnails are a v2 enhancement.
