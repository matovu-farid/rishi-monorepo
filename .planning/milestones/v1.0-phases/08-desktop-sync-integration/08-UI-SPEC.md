## Component Inventory

This phase adds 2 new UI components to the desktop app:

### 1. SyncStatusIndicator

**Location:** Sidebar footer area (below navigation items)
**Behavior:**
- Shows current sync state with icon + label
- States: idle (checkmark), syncing (spinning loader), error (alert triangle), offline (cloud-off)
- Click triggers manual sync when idle or in error state
- Tooltip on hover shows last sync timestamp: "Last synced: {relative time}"

**Visual spec:**
- Container: `flex items-center gap-2 px-3 py-2 rounded-md` using sidebar tokens
- Icon: 16x16 Lucide icon (`check`, `loader-2` with `animate-spin`, `alert-triangle`, `cloud-off`)
- Label: 12px medium weight, `var(--sidebar-foreground)` at 60% opacity for idle, full opacity for active states
- Hover: `var(--sidebar-accent)` background
- Click target: entire container row, min 32px height

### 2. SyncToast (uses existing toast/sonner pattern if available, otherwise inline)

**Behavior:**
- Auto-dismiss after 4 seconds for success, persist until dismissed for errors
- Position: top-right of viewport
- Shows on: first sync completion, sync error, migration complete

**Visual spec:**
- Success: `var(--card)` background, `var(--card-foreground)` text, green accent left border (4px)
- Error: `var(--card)` background, `var(--card-foreground)` text, destructive accent left border (4px)
- Max width: 360px
- Padding: 16px
- Border radius: `var(--radius)` (0.625rem)
- Shadow: `0 4px 12px oklch(0 0 0 / 10%)`

## Interaction Contracts

### Sync Trigger Points
| Trigger | Behavior | Visual Feedback |
|---------|----------|-----------------|
| App gains focus | Auto-sync in background | SyncStatusIndicator shows "Syncing..." |
| User clicks "Sync Now" | Immediate push/pull cycle | Spinner animation, disabled click during sync |
| Book imported | Dirty flag set, sync on next cycle | No immediate visual change |
| Reading position changed | Dirty flag set, sync on next cycle | No immediate visual change |
| Highlight created/edited/deleted | Dirty flag set, sync on next cycle | No immediate visual change |
| Periodic (every 5 min) | Auto-sync in background | SyncStatusIndicator updates silently |

### Sync Error Handling
| Error | User Sees | Recovery |
|-------|-----------|----------|
| Network offline | Indicator: "Offline" (cloud-off icon) | Auto-recovers when online, no user action needed |
| 401 Unauthorized | Toast: "Session expired. Please sign in again." | Redirect to sign-in |
| 500 Server error | Toast: "Sync failed. Check your connection and try again." + Indicator: "Sync failed" | User can click "Sync Now" to retry |
| Migration failure | Toast: "Library update failed. Please restart the app." | App restart triggers migration retry |

### State Transitions
```
not-synced -> syncing -> synced
                      -> error -> syncing (retry)
synced -> syncing (periodic/focus/manual) -> synced
any -> offline (network lost) -> syncing (network restored)
```
