<objective>
Fix two wiring bugs that break RAG, TTS, and embedding for synced books on mobile.

Purpose: Books synced from another device currently fail in the chat screen because (1) the chat screen uses getBookById (synchronous, no R2 download) instead of getBookForReading (async, triggers download), and (2) pipeline.ts throws when the on-device embedding model is not downloaded instead of falling back to the server endpoint.

Output: Synced books work correctly in chat -- file downloads on-demand, embedding uses server fallback when model unavailable, chat is usable without waiting for 80MB model download.
</objective>
