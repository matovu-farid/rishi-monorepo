1. Create `apps/mobile/lib/sync/file-sync.ts`:
   - Import `* as Crypto` from `expo-crypto`
   - Import `File, Directory, Paths` from `expo-file-system`
   - Import `apiClient` from `@/lib/api`
   - Import `db` from `@/lib/db`
   - Import `books` from `@rishi/shared/schema`
   - Import `eq` from `drizzle-orm`
   - Import types `UploadUrlRequest, UploadUrlResponse, DownloadUrlRequest, DownloadUrlResponse` from `@rishi/shared/sync-types`

   **Export `async function hashBookFile(filePath: string): Promise<string>`:**
   - Create a `new File(filePath)` using expo-file-system SDK 54 API
   - Read content as base64: `const base64 = file.base64()` (synchronous in SDK 54)
   - Hash using `await Crypto.digestStringAsync(Crypto.CryptoDigestAlgorithm.SHA256, base64)`
   - Return the hex hash string
   - NOTE: For very large files (50MB+) this may be slow. Acceptable for MVP -- books are typically 1-20MB.

   **Export `async function uploadBookFile(filePath: string, fileHash: string, format: 'epub' | 'pdf'): Promise<{ r2Key: string }>`:**
   - Determine contentType: `format === 'epub' ? 'application/epub+zip' : 'application/pdf'`
   - POST to `/api/sync/upload-url` via apiClient with body `{ fileHash, contentType }`
   - Parse response as UploadUrlResponse
   - If `response.exists === true`, return `{ r2Key: response.r2Key }` (dedup -- skip upload)
   - If `response.exists === false`:
     - Read file bytes: `const file = new File(filePath); const blob = await file.blob()` (or use fetch to read local file URI)
     - Upload to R2 using the presigned URL:
       ```typescript
       const uploadRes = await fetch(response.uploadUrl!, {
         method: 'PUT',
         body: blob,
         headers: { 'Content-Type': contentType },
       });
       ```
       NOTE: Do NOT use apiClient for the R2 upload -- the presigned URL goes directly to R2, not the Worker.
     - If upload fails, throw error
   - Return `{ r2Key: response.r2Key }`

   **Export `async function downloadBookFile(bookId: string, r2Key: string, format: 'epub' | 'pdf'): Promise<string>`:**
   - POST to `/api/sync/download-url` via apiClient with body `{ r2Key }`
   - Parse response as DownloadUrlResponse
   - Determine local destination: `documentDirectory/books/<bookId>/book.<format>`
   - Create directory if needed: `const bookDir = new Directory(Paths.document, 'books', bookId); bookDir.create({ intermediates: true, idempotent: true })`
   - Download file from R2 presigned URL:
     ```typescript
     const downloadRes = await fetch(response.downloadUrl);
     const blob = await downloadRes.blob();
     ```
   - Write to local file: Use `File.write()` from expo-file-system SDK 54 or save the blob
     ```typescript
     const destFile = new File(bookDir, `book.${format}`);
     destFile.write(blob);
     ```
   - Update the book record in DB with the local filePath:
     ```typescript
     db.update(books).set({ filePath: destFile.uri }).where(eq(books.id, bookId)).run();
     ```
   - Return `destFile.uri`

   IMPORTANT: All network calls must be wrapped in try/catch. Upload/download failures should throw so callers can handle gracefully (show error toast, retry later). The sync engine handles metadata; file-sync handles actual file transfer and is called explicitly during import and book-open flows.

   - Import `hashBookFile` and `uploadBookFile` from `@/lib/sync/file-sync`
   - Import `db` from `@/lib/db` and `books` from `@rishi/shared/schema` and `eq` from `drizzle-orm`

   In both `importEpubFile()` and `importPdfFile()`, AFTER the file is copied to local storage and AFTER `insertBook()` is called:
   - Hash the local file: `const fileHash = await hashBookFile(destFile.uri)`
   - Update the book record with the hash: `db.update(books).set({ fileHash, isDirty: true }).where(eq(books.id, bookId)).run()`
   - Upload to R2 in background (fire-and-forget with error logging):
     ```typescript
     uploadBookFile(destFile.uri, fileHash, 'epub').then(({ r2Key }) => {
       db.update(books).set({ fileR2Key: r2Key, isDirty: true }).where(eq(books.id, bookId)).run();
     }).catch((err) => {
       console.warn('Book upload failed (will retry on next sync):', err);
     });
     ```
   - The import function still returns the book immediately -- the upload happens in the background. The user can start reading right away.
   - Do the same for PDF import with format 'pdf'.

   NOTE: If upload fails (offline), the fileHash is still stored locally. On the next sync push, the server will have the book metadata with fileHash but no R2 key. A future enhancement could retry uploads for books that have fileHash but no fileR2Key. For MVP, the book will be re-uploaded when the user imports on another device (dedup prevents duplicate storage).

2. Add a `getBookForReading` function to `apps/mobile/lib/book-storage.ts`:
   - Import `downloadBookFile` from `@/lib/sync/file-sync`
   - Export `async function getBookForReading(id: string): Promise<Book | null>`:
     - Get book by ID using existing `getBookById`
     - If book is null, return null
     - If `book.filePath` is empty or the file does not exist at that path:
       - If `book.fileR2Key` exists: download the file with `await downloadBookFile(id, book.fileR2Key, book.format)`
       - After download, re-fetch the book (filePath is now updated by downloadBookFile)
       - If no fileR2Key, return book as-is (metadata-only, cannot read yet)
     - Return book with valid filePath

   This function should be called by the reader screens instead of `getBookById` when the user opens a book. The reader screens currently call `getBookById` -- they should switch to `getBookForReading` which handles the download-on-open case.

   NOTE: The download may take time for large books. The caller should show a loading indicator. For now, just making the function async is sufficient -- the reader screen already has a loading state.

Add a comment in book-storage.ts documenting this. The actual screen changes are minimal -- the reader screens already handle null/loading states.
